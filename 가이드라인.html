포트폴리오 수정 & 관리 가이드

이 문서는 index.html 파일 수정 방법과 GitHub Pages 배포 시 파일 관리 방법을 설명합니다.

1. GitHub 폴더 구조 (필수)

이미지와 PDF 파일을 정상적으로 불러오기 위해 아래와 같이 폴더를 만드세요.

[레포지토리 구조 예시] 
my-portfolio/ (최상위 폴더)
├── index.html        (메인 파일)
├── guide.html        (현재 보고 계신 파일)
└── assets/           (폴더 생성!)
    ├── my_photo.jpg  (내 사진)
    ├── project1.pdf  (프로젝트 보고서 PDF)
    ├── project2.png  (프로젝트 스크린샷)
    └── demo.pdf

2. 프로젝트 내용 수정하기

index.html 파일을 메모장이나 VS Code로 엽니다. 파일 맨 아래쪽 <script> 태그 내부를 보세요.

데이터 수정 예시

const projectDB = {
    "proj1": {
        title: "제목 수정",
        period: "날짜 수정",
        overview: "개요 내용을 여기에 적으세요.",
        goals: [
            "목표 1",
            "목표 2"
        ],
        // ... (중략)
        fileType: "pdf",              // pdf 또는 img (없으면 "")
        filePath: "./assets/demo.pdf" // assets 폴더 내 파일명
    },
    // ...
};

위 코드에서 filePath 부분에 ./assets/파일이름.확장자 형태로 적으면 모달창에 자동으로 뜹니다.

3. PDF 및 이미지 넣는 법

모달창 하단에 포트폴리오 상세 자료를 띄우고 싶다면:

  1. assets 폴더 안에 PDF나 JPG/PNG 파일을 넣으세요.
  2. index.html 스크립트의 해당 프로젝트 부분으로 이동합니다.
  3. fileType을 "pdf" 혹은 "img"로 변경합니다.
  4. filePath를 "./assets/파일이름.pdf"로 변경합니다.
주의: 한글 파일명은 오류가 날 수 있으니 영문 파일명(예: report_01.pdf)을 권장합니다.

4. HTML 텍스트(이름, 이메일 등) 수정

index.html 파일에서 Ctrl+F (찾기)를 눌러 수정하고 싶은 글자를 검색하세요.

  • 이름 수정: "채민석" 검색 후 변경
  • 이메일 수정: "minseok@gmail.com" 검색 후 변경
  • 소개글 수정: "About Me" 섹션 내부의 텍스트 변경

5. 배포 확인

수정한 파일을 GitHub에 Push하면 잠시 후 https://본인아이디.github.io/레포지토리명/ 에서 변경 사항을 확인할 수 있습니다.
